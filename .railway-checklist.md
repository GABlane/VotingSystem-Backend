# Railway Deployment Checklist

Quick checklist to deploy backend to Railway.

## Pre-Deployment

- [ ] Code is working locally (`npm run start:dev`)
- [ ] All tests pass (`npm run test`)
- [ ] Build succeeds (`npm run build`)
- [ ] `.env` file has all required variables
- [ ] Supabase database is set up
- [ ] Storage bucket `project-logos` created and public

## Push to GitHub

- [ ] Code committed to Git
- [ ] Pushed to GitHub repository
- [ ] Repository is accessible

## Railway Setup

- [ ] Railway account created
- [ ] New project created from GitHub repo
- [ ] Railway connected to correct repository

## Environment Variables Set

Copy from your local `.env` and paste into Railway:

- [ ] `SUPABASE_URL`
- [ ] `SUPABASE_SERVICE_ROLE_KEY`
- [ ] `SUPABASE_ANON_KEY`
- [ ] `JWT_SECRET` (use strong random string)
- [ ] `FRONTEND_URL` (temporary: http://localhost:3000)
- [ ] `PORT` (optional, Railway auto-assigns)

## Deployment

- [ ] First deployment triggered
- [ ] Build logs show success
- [ ] Runtime logs show "Nest application successfully started"
- [ ] Health check works: `curl https://your-app.up.railway.app`

## Generate Domain

- [ ] Domain generated in Railway settings
- [ ] Railway URL copied (e.g., https://your-app.up.railway.app)
- [ ] Test API endpoint: `curl https://your-app.up.railway.app/projects`

## Post-Deployment

- [ ] Test login endpoint
- [ ] Test creating a project
- [ ] Verify QR code generation works
- [ ] Check file upload works
- [ ] Update `FRONTEND_URL` after deploying frontend

## Security

- [ ] Changed default admin password
- [ ] Strong JWT_SECRET used
- [ ] Service role key not in frontend code
- [ ] CORS configured for production domain

## Optional

- [ ] Custom domain configured
- [ ] Monitoring/alerts set up
- [ ] Auto-deploy enabled
